# 任务：实现 Anyrouter 账号定时测活脚本（GitHub Actions）

## 背景

Anyrouter 是一个公益 Claude API 中转站，其调度策略疑似为账号级先来先用。有用户反馈：“如果有人想要验证一下自己的配置对不对，建议找个凌晨1点-凌晨5点这段时间，测试一下，甚至你可以写个脚本自动测试一下，这段时间大概率是能用的，其他时间不要瞎试了，纯纯浪费时间因为有时候早晨醒的早，5点爬起来用any，基本上能用到下午，好几天都是如此，但是如果上班时间开始用，基本上全天retry，怀疑是any的策略是账号级的先来先用，而不是请求级别的，我猜这么做应该是为了保证蒸馏数据的连续性”

目标是通过 GitHub Actions 定时对多个 Anyrouter token 发起轻量 Claude API 调用，
使账号在调度队列中保持活跃状态，从而在使用时获得更高优先级。

## 技术背景：Claude 配置文件机制

Claude Code CLI 通过读取 `~/.claude/settings.json` 获取 API 配置。
该文件为 JSON 格式，核心结构如下：

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "<token>",
    "ANTHROPIC_BASE_URL": ...
  }
}

字段说明：
- `ANTHROPIC_AUTH_TOKEN`：认证 token（Anyrouter 使用此字段）
- `ANTHROPIC_BASE_URL`：API 端点，指向 Anyrouter 的代理地址
- 写入时需保证原子性（先写临时文件，再 rename 替换）

只有token是在不同的请求中需要多配置并且更换的。下面是统一的配置模板：

### 2. GitHub Actions 工作流（.github/workflows/keepalive.yml）

要求：
- 定时触发：每天凌晨2:00开始，每50分钟触发一次，直到容器被销毁。（UTC+8，即 UTC 17:00开始）
- 为了避免多次触发容器，采用这样的实现：在凌晨2点的时候设置一个cron触发容器，然后在容器内部，每50分钟（这样是为了避免达到6小时上限）轮询一遍。也就是说一天只开一次容器，定时设在内部。
- 支持手动触发（workflow_dispatch）
- 运行环境：ubuntu-latest
- 从 GitHub Secrets 读取 token 列表（Secret 名称：ANYROUTER_TOKENS，格式为换行分隔的多个 token 字符串）
- 下载最新版本的claude code命令行工具
- 对每个 token 依次执行测活，token 之间间隔 30 秒
    - 为了避免被账号风控，对于测试的prompt，需要维护一些看起来是真实的工程需求短prompt，而不是简单的重复的你好
- 输出每个 token 的测试结果（成功/失败）。如果成功的话，发一封邮件反馈

不仅支持在工作流中定时运行，也要支持在本地运行、在本地单次和手动地运行。也需要有测试用例。

### 4. 重要实现细节

**Token 存储格式（GitHub Secret: ANYROUTER_TOKENS）**：
每行一个 token，例如：
```
sk-ant-xxx111
sk-ant-xxx222
sk-ant-xxx333
```

另外需要QQ邮箱、QQ邮箱smtp授权码的secret（可选，不填也没关系，那就简单测活）。用于向自己发送测活报告（每次只发一次，容器销毁前发，即时间到5h58min的时候发一次即可）

**工作流中读取并遍历 token**：
```yaml
- name: Run keepalive for all tokens
  env:
    ANYROUTER_TOKENS: ${{ secrets.ANYROUTER_TOKENS }}
  run: |
    echo "$ANYROUTER_TOKENS" | while IFS= read -r token; do
      [ -z "$token" ] && continue
      echo "Testing token: ${token:0:10}..."
      bash scripts/keepalive.sh "$token" "https://anyrouter.top/v1"
      sleep 30
    done
```
注意随机时延。

**原子写入 settings.json 的 bash 实现**：
```bash
SETTINGS_DIR="$HOME/.claude"
SETTINGS_FILE="$SETTINGS_DIR/settings.json"
TMP_FILE="$SETTINGS_DIR/settings.json.tmp.$(date +%s%N)"

mkdir -p "$SETTINGS_DIR"

cat > "$TMP_FILE" << EOF
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "$TOKEN",
    "ANTHROPIC_BASE_URL": "$BASE_URL"
  }
}
EOF

mv "$TMP_FILE" "$SETTINGS_FILE"
```
